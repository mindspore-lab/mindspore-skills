# Demo: Qwen3 0.6B PTA New Workspace

## Goal

Demonstrate a real `readiness-agent` workflow:

- the user provides a brand-new workspace
- the user has already installed the NPU driver and CANN
- the agent helps add `uv`, `.venv`, PTA dependencies, a training script, the
  model, and the dataset
- the workflow ends with a `READY` certification and a prompt asking whether to
  run the real script

This is the first formal `readiness-agent` demo under `examples/`.

## Prerequisites

- the machine already has the NPU driver and CANN installed
- the user has already sourced the correct CANN environment script
- Hugging Face is reachable, or a mirror is configured
- the current directory is brand-new, or at least does not already contain a
  training script and `.venv`
- the user wants to train `Qwen/Qwen3-0.6B` through the `PTA` route

## Recommended Starting Directory

```bash
mkdir example_readiness_case
cd example_readiness_case
source /path/to/cann_env.sh
```

Optional environment variable:

```bash
export HF_ENDPOINT=https://hf-mirror.com
```

If `HUGGINGFACE_HUB_CACHE` or `HF_HOME` is not set, the agent may place model
and dataset caches near the current workspace by default.

## Recommended Prompt

```text
/fix Check whether this new workspace can train Qwen/Qwen3-0.6B with PTA. If the virtual environment, dependencies, training script, model, or dataset are missing, repair them inside the current workspace and tell me whether the workspace is READY after readiness completes.
```

If you want to specify the dataset in the first turn, use:

```text
/fix Check whether this new workspace can train Qwen/Qwen3-0.6B with PTA using karthiksagarn/astro_horoscope as the dataset. If the virtual environment, dependencies, training script, model, or dataset are missing, repair them inside the current workspace and tell me whether the workspace is READY after readiness completes.
```

## Example Interactive Follow-Up

If the agent asks whether it should apply repairs, answer with:

```text
Repair the environment.
```

After readiness returns `READY`, use:

```text
Run Qwen3 training.
```

## Expected Agent Actions

1. Identify the target as `training` and the framework as `PTA`.
2. Check whether `.venv`, a training script, the model, and the dataset already
   exist in the current workspace.
3. If anything is missing, install `uv` into user space and create a
   workspace-local `.venv`.
4. Install runtime dependencies inside `.venv`, such as `torch`, `torch_npu`,
   `transformers`, and `datasets`.
5. If the workspace does not contain a training script yet, generate a minimal
   runnable script.
6. Download `Qwen/Qwen3-0.6B` and the declared dataset.
7. Run `runtime_smoke`.
8. Generate:
   - `readiness-output/report.json`
   - `readiness-output/report.md`
   - `readiness-output/meta/readiness-verdict.json`
   - `.readiness.env`
9. If the verdict is `READY` or `WARN`, ask:
   `Do you want me to run the real model script now?`

## Expected Artifacts

The workspace will usually contain:

- `.venv/`
- `train.py`, or an equivalent training script generated by the agent
- `model/`, or a Hugging Face cache location
- `dataset/`, or a Hugging Face cache location
- `readiness-output/`
- `.readiness.env`

## Success Criteria

This demo is successful when:

- `readiness-output/meta/readiness-verdict.json` reports `status: READY`
- `can_run` is `true`
- `next_action` explicitly asks whether to run the real model script
- the generated training script, model, and dataset all live in the current
  workspace or in a cache location explicitly chosen by the user

## Boundary Notes

- This demo does not require `readiness-agent` to install drivers, firmware, or
  CANN.
- If the user has not sourced the correct CANN environment first, the agent may
  return `WARN` or `BLOCKED`, but it should not invent a fake system-level
  repair.
- If multiple CANN versions exist on the machine, sourcing the intended one in
  advance helps the agent avoid validating against the wrong runtime.
- If the user explicitly wants the `MindSpore` route, switch to a dedicated
  MindSpore demo. This demo only covers `PTA`.
- State the intended framework clearly. Otherwise the agent may need to infer
  it from the current workspace evidence.
- If `HUGGINGFACE_HUB_CACHE` is already set, model and dataset downloads will
  use that cache location instead of the current working directory.
- If the user specifies only the `Qwen3` family and the workspace does not
  already contain a runnable script, the agent may still need the exact Hugging
  Face model id and dataset choice before it can finish the repair path.
